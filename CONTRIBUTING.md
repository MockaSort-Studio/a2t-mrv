# Contributing

## Requirement traceability

Every test and CI step that covers a requirement must carry a `@req:` annotation linking it to the requirement ID.

```
@req: CRCF-28
```

A test covering multiple requirements takes one annotation per requirement. Apply it in the language's native syntax:

**Elixir**
```elixir
@tag req: REQ-ID
#or
# @req: REQ-ID
test "duplicate submission is rejected" do
  ...
end
```

**Python**
```python
# @req: REQ-ID
def test_duplicate_submission_rejected():
    ...
```

**GitHub Actions**
```yaml
- name: Validate content hash uniqueness
  # @req: REQ-ID
  run: mix test --only crcf_28
```

The traceability report is generated by running:

```bash
grep -r "@req:" .
```

Do not maintain it manually — it is always derived from the annotations in the code.
